Maven依赖

发表于 2019-01-06 | 分类于 maven | 阅读次数:

  Maven依赖小结，来源Maven实战

依赖范围

1. compile默认范围，编译测试运行都有效
2. provided编译测试有效，运行时不会被加入。如servlet api运行时由tomcat容器提供
3. runtime在测试与运行时有效
4. test在测试时有效
5. system与本机系统相关，范围同provided
6. import导入的范围，只在dependencyManagement中，表示从其他pom中导入dependency配置

依赖传递

  A依赖B,B依赖C。此时A对于B是第一直接依赖，B对于C时第二直接依赖，A对于C是传递性依赖。

1. 第二直接依赖为compile,对应传递性依赖为第一直接依赖
2. 第二直接依赖为test,对应传递性依赖不存在
3. 第二直接依赖为provided,只有第一直接依赖为provided时，传递性依赖为provided
4. 第二直接依赖为runtime,第一直接依赖为compile时传递性依赖为runtime,其他为第一直接依赖

发表于 2019-01-01 | 分类于 mybatis | 阅读次数:

  分析下mybatis这个半orm框架基本的执行流程。

1. 解析配置文件，生成全局的Configuration
2. 通过Configuration构建DefaultSqlSessionFactory.
3. 通过SessionFactory获取SqlSession,SqlSession主要生成了一个Executor
4. SqlSession.getMapper-> Configuration.getMapper->mapperRegistry.getMapper->MapperProxyFactory.newInstance生成代理对象返回
5. 执行相应的接口方法转到MapperProxy.invoke->MapperMethod.execute->选择执行DefaultSqlSession中对应方法，从Configuration获取MappedStatement->获取BoundSql
6. 最终转到Executor中，拿到Connection预编译sql,设置参数，执行sql获取返回的结果封装成对应的Bean返回。

JAVA正则类Pattern,Matcher小结

发表于 2018-09-28 | 分类于 java | 阅读次数:

  Java中正则表达式分别要用到java.util.regex.Pattern和java.util.regex.Matcher类。其中Pattern通过编译正则表达式创建一个匹配模式,Matcher通过Pattern实例提供的正则表达式对字符串进行匹配。
  在这里对正则表达式不做说明，可通过30分钟正则表达式入门。一般可通过RegexBuddy等工具先学习测试正则表达式。

Pattern类

1. Pattern p=Pattern.compile("\\d+") ＃通过Pattern的静态方法compile创建Pattern对象。
2. Matcher m=p.matcher(input) #为目标字符串input创建一个Matcher对象

Matcher对象

常用方法说明

1. matches()匹配整个输入
2. lookingAt()只匹配输入开始的字符串
3. find()对整个字符串进行匹配,可能匹配多处
4. 3个方法均返回布尔值true或者false
5. 执行matches(),lookingAt(),find()匹配操作后，可以利用start(),end(),group()得到更详细的匹配信息。
6. start()返回匹配到的子字符串在字符串中的索引位置. 
7. end()返回匹配到的子字符串的最后一个字符在字符串中的索引位置. 
8. group()返回匹配到的子字符串
9. groupCount()返回有几个正则分组
10. start(i),end(i),group(i)返回第几个分组匹配
11. reset()重置到字符串开始匹配
12. replaceAll()和replaceFirst()方法可以用于替换Matcher搜索字符串中的一部分。replaceAll()方法替换全部匹配的正则表达式，replaceFirst()只替换第一个匹配的。
